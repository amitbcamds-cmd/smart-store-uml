Object Definition

Class Diagram: Exam Theory

Key points to remember for your university exams.

Static Structural View

  • It represents the static view of an application.
  • It describes the structure of a system by showing the system's classes, attributes, operations, and relationships.
  • Class diagrams are the only UML diagrams which can be mapped directly with object-oriented languages.

Class Compartments

  • Top Compartment: Contains the name of the class (Centered, Bold, Starts with uppercase).
  • Middle Compartment: Contains the attributes (Fields/Variables) that describe the state.
  • Bottom Compartment: Contains the operations (Methods/Functions) that describe the behavior.

Visibility Modifiers

  • + (Public): Visible to all elements in the system.
  • - (Private): Visible only within the class itself.
  • # (Protected): Visible to the class and its descendants (subclasses).
  • ~ (Package): Visible to elements within the same package.

Multiplicity Rules

  • Multiplicity defines how many instances of one class are linked to one instance of another class.
  • Common notations: 0..1 (Optional), 1 (Exactly one), * (Many), 1..* (One or more).
